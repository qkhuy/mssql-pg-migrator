# migrator-ui (Wails desktop app)

A desktop UI for the migrator tool, built with [Wails](https://wails.io)
(Go backend + Svelte frontend). It binds the **same** `internal/app.Service`
the CLI uses, so the UI and CLI behave identically.

This is a **separate Go module** (`ui/go.mod`) with a `replace` directive to the
core. That keeps the core's `go build ./...` and CI independent of the GUI
toolchain (Wails needs a C/WebView toolchain that isn't required for the CLI).

## Prerequisites

- Go 1.25+
- Node.js 18+ (frontend build only)
- The Wails CLI: `go install github.com/wailsapp/wails/v2/cmd/wails@latest`
- Platform WebView:
  - Windows: WebView2 runtime (preinstalled on Win10/11)
  - macOS: Xcode command-line tools
  - Linux: `gtk3` + `webkit2gtk` dev packages (e.g. `libgtk-3-dev libwebkit2gtk-4.0-dev`)

Run `wails doctor` to verify your environment.

## Develop

```bash
cd ui
wails dev
```

`wails dev` generates the JS bindings under `frontend/wailsjs/` (the
`Engines/Assess/Plan/Run` calls and the `runtime` event helpers that
`src/App.svelte` imports), builds the Svelte frontend, and launches the app with
hot reload.

## Build a native binary

```bash
cd ui
wails build      # output in ui/build/bin/
```

## Screens / flow

Connection (source & target DSN) → **Assess** (mappings + status) →
**Plan** (dry-run DDL) → **Run** (live per-table progress via the `progress`
event) → result. All operations are the service-layer methods in
`internal/app`, exposed by `ui/app.go`.

> Note: `frontend/wailsjs/` is generated by Wails and is intentionally not
> committed; it appears after your first `wails dev`/`wails generate module`.
